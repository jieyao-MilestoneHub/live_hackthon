## 逐頁大綱（12 頁）

### 1. 封面
- 標題：**浪 LIVE — AI 直播高光偵測 + 剪輯短片編輯器**
- 一句話價值：*「上傳直播，AI 幫你剪出高光短片；你只需微調，一鍵產出。」*
- 視覺：產品名 + 一張成品短片縮圖（P4 素材）。
- 講稿：「我們解決的是——直播主播完，最有價值的高光片段卻埋在幾十分鐘素材裡。」

### 2. 問題
- 直播/長影音創作者痛點：
  - 一場直播 30–120 分鐘，人工找高光 + 剪輯耗時（常 1–3 小時）。
  - 短影音平台（Reels/Shorts/TikTok）需要**豎版、≤60 秒、有字幕與節奏**。
  - 剪輯門檻高，錯過發佈黃金時間。
- 講稿：「痛點不是『不會剪』，而是『太慢』——內容的時效性被剪輯速度綁架。」

### 3. 解法概述
- 三句話：
  1. **AI 自動偵測高光**（逐字稿 + 規則/LLM 評分）。
  2. **自動組出符合目標秒數的初始草稿**（Composer，毫秒精度、不超過 60s）。
  3. **可編輯**：使用者調整順序/字幕/特效後，一次渲染成品。
- 差異化：**兩段式**——不是黑盒自動剪，而是「AI 出草稿 + 人類把關」。
- 視覺：P1 的「三步精簡版」流程圖。

### 4. 使用者流程（取自 P1）
- 放 P1 端對端流程圖（分析段 + 編輯段）。
- 強調三個關鍵設計：
  - 分析工作流**產出草稿即結束**，不卡等使用者（可隔天再回來）。
  - Timeline **版本化 append-only**，可 Undo / 重 Render。
  - 特效用 **`effect_seed`** 保證重試可重現。

### 5. 三大模組設計
> 對應團隊三工作流：analysis（分析）/ system（系統）/ presentation（簡報）；技術上是三個產品模組。

| 模組 | 職責 | 技術亮點 |
|------|------|----------|
| ① 智慧分析 | 逐字稿 → 高光候選（start/end/score/reason/title） | 規則式評分 + 可選 Bedrock 標題 enrich；契約 `highlights.v1` |
| ② 自動剪輯合成 | 高光 → 目標秒數 Timeline → FFmpeg 成品 | 貪婪填秒數、避免句中切、`effect_seed` 可重現；一次編碼（不重複轉檔） |
| ③ 互動編輯器 | 建 Project、上傳、四區編輯、Render、下載 | Next.js 靜態匯出 + Cognito；狀態機驅動 UI |

### 6. AWS 架構圖（取自 S1 / #8）
- 放**架構現況圖**（帳號 979287128595 / us-east-1）。
- 一句話：**全無伺服器控制面 + 事件驅動分析管線 + Batch 重型渲染**。
- 標註 workshop 限制的務實取捨（見第 8 頁）。

### 7. 技術亮點（工程可信度）
- **Serverless 控制面**：API Gateway HTTP API + Lambda（Mangum 容器），零常駐成本。
- **事件驅動分析**：S3 → EventBridge → SQS(+DLQ) → Starter → Step Functions，冪等、可重試、可觀測。
- **契約驅動協作**：`contracts/` 為單一真相，schema 版本化（Project/毫秒模型 v0.2.0）。
- **毫秒精度 + 可重現**：時間全 ms；`effect_seed` 固定隨機特效。
- **IaC**：Terraform 遠端 state（S3 + DynamoDB lock），單一 dev 環境。

### 8. 務實工程取捨（加分項：懂雲端限制）
- Workshop 帳號 **SCP 擋 App Runner 與公開 Lambda Function URL** → 改用 **API Gateway HTTP API + Lambda** 達成公開控制面。
- 「每加一個新 AWS 服務先小 probe（create→delete）確認 SCP 允許」→ 展現嚴謹。
- 輸入/輸出分桶避免事件迴圈（S3 事件至少一次投遞 → Starter 以 `bucket+key+version_id` 冪等）。

### 9. 成效衡量（取自 P3）
- 五大指標：剪輯時間節省比例、高光偵測準確率、短片品質評分、自動化程度、商業效益。
- 放 P3 的指標表（目標值 + `〈待 #14 實測〉` 欄）。
- 講稿：「指標定義與量測方法已就緒，端到端實測數字回填後即可佐證。」

### 10. Demo
- 現場走 P6 腳本（或播 P7 備援錄影）。
- 一句過場：「接下來實際操作一次：上傳 → 看 AI 高光 → 微調 → 產出短片。」

### 11. 里程碑與現況
- M0 走路骨架 ✅ 已部署上線（前端 CloudFront + 後端 API GW/Lambda 端到端驗證）。
- M1 兩段式編輯器契約/控制面 ✅（10 個 Editor API 端點、103 測試綠）。
- M2 事件驅動分析管線 🔨 進行中（Step Functions + SQS + IAM 接線）。
- M3/M4 渲染面（Creative + FFmpeg/Batch）🔨 接縫就緒。
- 誠實呈現：不誇大「全自動上線」，標清進行中項目。
