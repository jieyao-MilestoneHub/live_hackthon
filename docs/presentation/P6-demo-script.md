## 二、現場操作腳本（步驟 + 話術）

> 全程約 3–4 分鐘。**話術**為建議口播，**動作**為畫面操作，**⚠️** 為易失敗點與備援。

### 步驟 1：進入產品、說明價值（20 秒）
- **動作**：開前端首頁。
- **話術**：「這是浪 LIVE。假設我剛結束一場 30 分鐘直播，想馬上發一支高光短片到 Reels。」

### 步驟 2：建立 Project、指定秒數（20 秒）
- **動作**：輸入標題 → 目標秒數設 **30 秒** → 建立。
- **話術**：「我只要告訴它：我要一支 30 秒的短片。其餘交給 AI。」
- **⚠️**：若建立失敗 → 多半是登入/JWT 過期。**備援**：重新登入；再不行切錄影。

### 步驟 3：上傳素材（30 秒）
- **動作**：選檔 → 瀏覽器直傳 S3（presigned multipart）→ 顯示上傳進度。
- **話術**：「影片直接從瀏覽器上傳到 S3，不經過我們的伺服器，快又省。」
- **⚠️ 最大風險點**：大檔上傳慢 / multipart complete 握手。**備援**：改用**預先上傳好的 Project**（見步驟 4 直接跳轉）。

### 步驟 4：AI 分析 → 高光候選（40 秒）
- **動作**：狀態列走 `ANALYZING → COMPOSING → READY_TO_EDIT`；編輯器出現高光候選 + 初始 Timeline。
- **話術**：「AI 轉了逐字稿、找出高光時刻，還自動組了一版符合 30 秒的草稿——注意，它沒有幫我決定一切，而是給我一個**可編輯的起點**。」
- **⚠️**：Transcribe 需時間（真實幾十秒~數分）。**備援**：直接開**已 `READY_TO_EDIT` 的預備 Project**（demo 用兩個 Project：一個現場上傳、一個已備好）。

### 步驟 5：微調 Timeline（30 秒）
- **動作**：刪一個片段 / 調順序 / 鎖定一個高光 / 開字幕 / 選 9:16 比例。
- **話術**：「我可以刪掉不要的、鎖定一定要留的、選豎版比例、開字幕——每次修改都是新版本，可以隨時回復。」

### 步驟 6：渲染並下載（40 秒）
- **動作**：提交 Render → 狀態走 `RENDERING → ARTIFACT_READY` → 下載 `final.mp4`（或播 preview）。
- **話術**：「按下渲染，FFmpeg 一次完成裁切、字幕、特效、編碼。幾分鐘前還是 30 分鐘的直播，現在是一支可以直接發佈的短片。」
- **⚠️**：Batch 渲染需時間。**備援**：播放 P7 預錄的成品短片。

### 收尾（15 秒）
- **話術**：「AI 出草稿，創作者做主——這才是真正可用的自動剪輯。」

## 三、易失敗環節 × 備援切換總表

| 環節 | 風險 | 徵兆 | 備援切換 |
|------|------|------|----------|
| 登入 | JWT 過期 | 建 Project 401 | 重新登入 → 切錄影 |
| 上傳 | 大檔/握手慢 | 進度卡住 | 跳用預備 Project |
| Transcribe | 耗時長 | 卡 `ANALYZING` | 開已備好的 `READY_TO_EDIT` Project |
| Batch 渲染 | 冷啟/耗時 | 卡 `RENDERING` | 播 P7 成品短片 |
| 網路 | 會場 Wi-Fi 不穩 | 全面變慢 | 切手機熱點 |
| 全面失敗 | 任一環節崩 | — | **直接切 P7 完整錄影** |

> 原則：**任何一步卡超過 15 秒，主持人不解釋 bug，直接切備援錄影繼續講。**

## 五、P7 備援 Demo 錄影分鏡（#21）

> 目標：完整預錄一次**成功**流程，現場任何環節失敗時可替代。錄製時關掉通知、用乾淨 demo 帳號、螢幕 1080p。

| 分鏡 | 畫面 | 旁白（同 P6 話術） | 時長 |
|------|------|--------------------|------|
| 1 | 首頁 | 產品價值開場 | 0:00–0:20 |
| 2 | 建 Project 設 30s | 「我要一支 30 秒短片」 | 0:20–0:40 |
| 3 | 上傳進度 | 直傳 S3 說明 | 0:40–1:10 |
| 4 | 狀態走到 READY_TO_EDIT + 高光 | AI 分析 + 可編輯起點 | 1:10–1:50 |
| 5 | 微調 Timeline | 刪/鎖/比例/字幕 | 1:50–2:20 |
| 6 | 渲染 → 成品下載 → 播放短片 | FFmpeg 一次成品 | 2:20–3:10 |
| 7 | 收尾金句 | 「AI 出草稿，創作者做主」 | 3:10–3:25 |

**錄製檢查**：
- [ ] 全程無失敗、無 401/卡頓。
- [ ] 成品短片實際播放 3–5 秒入鏡。
- [ ] 存兩份（本機 + 雲端），現場播放器先測過。
- [ ] 準備「無旁白純畫面版」以便主持人現場口播。
